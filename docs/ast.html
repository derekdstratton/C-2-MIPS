Abstract Syntax Tree

The Abstract Syntax Tree provides an intermediate representation between the Parser's productions and the Three Address Code. Multiple nodes of various types are built by the Parser, and will later be traversed to generate the code. Currently there are 12 subclasses of ASTNode, which is the main base node that every node in the Abstract Syntax Tree is derived from. TypeNode is an important subclass of ASTNode, as it has 10 subclasses that represent nodes with some sort of type. More information can be found on the ASTNode class page. The tree is stored in the non-binary tree class tree.hh, which has been modified in tree_util2.hh to print to a file for output that can be viewed in GraphViz.

Testing the Abstract Syntax Tree

The AST can be tested by running the Test Files with ScannerParser. The program argument should contain a file from the tests folder.

  • https://github.com/derekdstratton/C-2-MIPS/tree/master/tests

When running, add the program argument -a to output the AST to a file. These tests can be compared with their outputs, found in the output directory.

  • https://github.com/derekdstratton/C-2-MIPS/tree/master/tests/output_ast
